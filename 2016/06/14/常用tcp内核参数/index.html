常用tcp内核参数

ip报文格式

tcp报文格式

连接过程相关

somaxconn

listen()系统调用可以设置的backlog限制值

tcp_max_syn_backlog

处于半连接状态（还没有收到客户端ack的连接）的连接的最大数量

tcp_synack_retries

三次握手过程中，server没有收到客户端最后一次ack时，默认会重发syn-ack包，tcp_synack_retries控制重发次数，默认是五次。由于每次重试间隔翻倍（即1s, 2s, 4s, 8s, 16s），算上等待最后一次超时的32s，server会保持半连接状态的连接63s

tcp_syn_retries

第一个syn的重试次数，默认为6次，与tcp_synack_retries类似，6次间隔为1s, 2s, 4s, 8s, 16s, 32s，因此最后一次重发是在63s之后，而最终的超时在127s之后

tcp_abort_on_overflow

一旦出现server无法接收新的请求，重置所有client。Enabling this option can harm clients of your server

tcp_syncookies

内核编译时加上CONFIG_SYN_COOKIES才生效。当syn_backlog溢出后，server会发送一个由源地址端口、目标地址端口和时间戳构造的syn_cookie发送给客户端，客户端把syn_cookie返回后，连接就可以建立，不再需要进入syn_backlog。

这个参数违反了tcp协议，在高负载系统中不建议使用。当出现syn_flood攻击时，推荐设置tcp_max_syn_backlog, tcp_synack_retries, tcp_abort_on_overflow参数

buffer设置

net.core.rmem_max和net.core.wmem_max

setsockopt可设置的缓冲区最大值

####tcp_rmem和tcp_wmem

min, default, and max number of bytes。如果通过setsockopt设置了缓冲区大小，则max值被忽略

tcp_moderate_rcvbuf

receive buffer自动调整，默认开启

和断开连接相关

####tcp_fin_timeout

处于FIN_WAIT_2状态的超时时间，默认是60s

tcp_tw_reuse

tcp_timestamps开启才生效，且只能保证outgoing connection能够快速重用time_wait连接

tcp_tw_recycle

tcp_timestamps开启才生效。对于nat网络，可能导致connect fail情况。 例子

tcp_max_tw_buckets

系统最大time_wait连接数量，尽量增大这个数量

tcp_max_orphans

系统所能处理不属于任何进程的TCP sockets最大数量

tcp_retries1 和 tcp_retries2

经过tcp_retries1次重传后，tcp会把错误汇报给网络层，可能影响路由选择。默认值是3，RFC 1122推荐这个值

已经建立的连接，经过tcp_retries2次重传，连接直接关闭。默认是15次，超时时间达到15分钟；RFC 1122推荐超时时间至少100s，因此推荐8次

RFC 1122关于这种情况有如下描述：

`4.2.3.5 TCP Connection Failures

Excessive retransmission of the same segment by TCP
indicates some failure of the remote host or the Internet
path. This failure may be of short or long duration. The
following procedure MUST be used to handle excessive
retransmissions of data segments [IP:11]:

(a) There are two thresholds R1 and R2 measuring the amount
of retransmission that has occurred for the same
segment. R1 and R2 might be measured in time units or
as a count of retransmissions.

(b) When the number of transmissions of the same segment
reaches or exceeds threshold R1, pass negative advice
(see Section 3.3.1.4) to the IP layer, to trigger
dead-gateway diagnosis.

(c) When the number of transmissions of the same segment
reaches a threshold R2 greater than R1, close the
connection.

(d) An application MUST be able to set the value for R2 for
a particular connection. For example, an interactive
application might set R2 to “infinity,” giving the user
control over when to disconnect.

(e) TCP SHOULD inform the application of the delivery
problem (unless such information has been disabled by
the application; see Section 4.2.4.1), when R1 is
reached and before R2. This will allow a remote login
(User Telnet) application program to inform the user,
for example.`

其中R1对应tcp_retries1，R2对应tcp_retries2

tcp_orphan_retries

对于本地已经关闭的连接，重传超过tcp_orphan_retries次后，关闭连接。默认值是8。对于高负载的web服务器，应该降低这个值

tcp option相关

tcp_window_scaling

开启window scaling。在tcp协议中，tcp receiver window占用2个字节，即最大可以设置为65,535，这个大小明显是不够的。因此在tcp head option字段中，可以使用一个byte记录一个位移位数，当计算tcp receiver window时，需要左移这个位数才能计算出真正的窗口值。这个位移位数最大可以设置为14，即窗口可以最大为2^(16+14) = 1G 见window_scaling

tcp_sack

开启sack（select acknowledgments）。当出现包未接受到时，接收方会记录已经收到包序号，并且记录到sack中发回给发送方。如此，发送方就可以确认重发的包序号

tcp_dsack

允许duplicate sack。使用sack告知发送方哪些数据被重复发送了。

tcp_fack

开启fack。fack是一种使用sack的拥塞避免快速重传算法

####tcp_timestamps
开启tcp_timestamps。在option中加入两个四字节的时间，第一个是本机本次发送的timestamp，第二个是上一次从对端发送过来数据的timestamp。

tcp_timestamps主要有三个用处：序列号回绕，乱序的时间判断依据、避免确认二义性，以及计算RTT

tcp keepalive

####tcp_keepalive_intvl

每次keepalive探测包之间的时间间隔，默认75s，对应socket option：TCP_KEEPINTVL

####tcp_keepalive_probes

keepalive探测报探测次数，默认9次，对应socket option：TCP_KEEPCNT

tcp_keepalive_time

连接空闲tcp_keepalive_time时间后，会开始keepalive探测，默认是2小时，对应socket option：TCP_KEEPIDLE。只有在socket option中开启了 SO_KEEPALIVE才会生效。
